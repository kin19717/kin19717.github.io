Cloud virtualized IP Multimedia Subsystem (vIMS)

    Openstack Linux Yaml Json Xml Python

• The lead integrator and coordinator of the first NAR customer Cloud deployments of SCCAS(NTAS) (Nokia Telecom Application Server platform) and BRDB (Redis Platform) for the AT&T ECOMP environment • Base on the large data set that AT&T (Gold Standard) provides, I weekly update the Cloud Service Archive (CSAR) on GitHub to meet the requirements of AT&T Master Service Orchestrators (MSO) compliance and to align the configuration data into this CSAR. • Modify the CSAR in TOSCA specification to standardize how AT&T describes software applications and the infrastructure components that are required for them to run in the cloud. • Create python and shell script to automate Linux command, package CSAR with configuration files (YAML and JSON) and deploy the CSAR into the OpenStack cloud orchestration layer for a lifecycle from creation to termination by using Redhat Linux. • Run ansible playbook to bring up the application server GUI, monitor the traffic and alarms and execute configuration file (XML) into the system to bring up the containers. • Create Method of Procedure test environment, execute a test plan to audit all configuration data in the system before delivering the CSAR to the customer. • Validate the procedures with AT&T engineers during maintenance window and support services after the CSAR deployed in the customer cloud data centers.
